AI-Based Collaborative Study System - Full Frontend Design and Route Guide

Web Design and Look
the website must look very professional and premium.
it will use a dark theme (dark charcoal and black) so it looks cool and is easy to see.
we will use a "glass" look for all matching boxes and cards. they will be a bit see-through with a blur.
animations are very important. when you click a button or move to a new page, it should be smooth.
the fonts should be modern and clean like 'Inter'.
the goal is a "wow" feeling for any user who opens the website.

Login and Signing Up
users have two ways to login. 
1. you can use your google account.
2. you can use email and password.
if you sign up with email, you need to enter a username, your email, and a password (at least 6 letters long).
to find out who is logged in, the frontend calls /auth/me. it gives you the userid and email.
when you want to leave, just click the logout button to clear everything.

Friends and Users
to add a new friend, use api/friend-request/send.
to see who your friends are and your friend count, use api/get-friends.
to accept or reject a request someone sent you, use api/friends-request/action.
to see all the requests you received from others, use api/get-reqests.
to see every person on the system, use api/get-users.
to find a specific person by name, use GET api/search-users. 
examples:
GET /search-users?username=john (finds john)
GET /search-users?username=john&offset=10 (skips first 10 results)

Notifications and Page Data
to see new alerts you haven't seen yet, use api/get-notifications/unseen.
to see all your old alerts, use api/get-notifications/history.
the types of alerts you might see are:
- friend_request (someone wants to be your friend)
- friend_request_accepted (they said yes!)
- friend_request_rejected (they said no)
- tried_to_reach_out (someone sent you a private message while you were offline)
- group_message (someone sent a message in your study group)

the home page with all the main info uses api/pages/home.
your personal dashboard with all your stats uses api/pages/dashboard.

AI Smart Study Tools
to see all the smart things the AI can do, use api/ai/available-options.
to see what topics you can take a quiz on, use api/ai/quiz/available-options.
to get a quiz prepared by AI, use api/ai/get-quiz.
to send your quiz answers for checking, use api/ai/post-quiz.
to get quick flashcards for any topic, use /api/ai/flashcards.
to talk to the AI assistant for help, use POST /api/ai/chat.
to get a short summary of any notes you have, use /api/ai/summary.
to get the most important questions for an exam, use /api/ai/important-questions.

Notes
for notes there are exactly 6 apis.
users can only type text for notes. no file uploads allowed here.
you can create a note, see all notes, see one specific note, search for notes, edit notes, and delete them.
all these are under the /api/notes section.
possible values you can use:
- topicImportance: high, medium, low
- isArchived: true, false (use true to delete/archive)

Resources (Files and Documents)
this is the special place for uploading files like PDFs.
you can upload a new file, share any file with a friend, see all your files, or delete them.
everything uses the /api/resources route.

Study Groups
you can make a study group for any subject and invite other students.
you can search for groups, see all existing groups, or see one group's full info.
if you are the owner, you can edit the group name/details or delete the group.
private groups are locked, and people must ask to join.
possible values:
- isPrivate: true, false

Study Group Members
you can join public groups or leave any group you want.
you can see the full list of everyone in a group.
the owner can make a member an admin or take it back.
the owner or admin can remove any member from the group.
possible values:
- role: owner, admin, member

Join Requests
for private groups, you have to click "ask to join". this uses /groups/:groupId/request.
the group admin sees all requests at /groups/:groupId/requests.
the admin can approve or reject the request. you only get in if they say yes.
possible values:
- status: pending, approved, rejected

Group Messages (Chat)
every group has a real-time chat box.
you can send messages to the whole group.
you can see everything people said before in the history.
the chat works instantly using sockets.

================================================================================
COMPLETE API ROUTE REFERENCE
================================================================================

1. Authentication (/auth)
- GET /auth/google - Login with Google.
- GET /auth/google/callback - Google Callback.
- GET /auth/me - Get my info.
- POST /auth/register - JSON Body: { "username", "email", "password" }
- POST /auth/login - JSON Body: { "username", "password" }
- POST /auth/logout - Logout.

2. Study Groups (/api/groups)
- GET /api/groups/search?q=query - Search groups.
- GET /api/groups/ - List all.
- GET /api/groups/:id - Single group info.
- POST /api/groups/ - Create. Body: { "name", "description", "isPrivate" }
- PUT /api/groups/:id - Update.
- DELETE /api/groups/:id - Delete.

3. Study Group Members (/api/group-members)
- GET /api/group-members/my-groups - Groups I joined.
- GET /api/group-members/:groupId/members - List members.
- POST /api/group-members/:groupId/join - Join group.
- PUT /api/group-members/:groupId/promote/:userId - Make admin.
- PUT /api/group-members/:groupId/demote/:userId - Remove admin.
- DELETE /api/group-members/:groupId/leave - Leave group.
- DELETE /api/group-members/:groupId/remove/:userId - Kick member.

4. Group Messages (/api/groups/:groupId/messages)
- GET /api/groups/:groupId/messages - Get chat history.

5. Join Requests (/groups)
- GET /groups/:groupId/requests - See pending requests.
- POST /groups/:groupId/request - Ask to join.
- PUT /groups/:groupId/requests/:userId/approve - Admin approves.
- PUT /groups/:groupId/requests/:userId/reject - Admin rejects.

6. Notes (/api/notes)
- GET /api/notes/search - Search notes.
- GET /api/notes/ - List user notes.
- GET /api/notes/:id - Single note info.
- POST /api/notes/ - Create. Body: { "name", "subject", "topic", "content" }
- PUT /api/notes/:id - Update note.
- DELETE /api/notes/:id - Remove note.

7. Resources (/api/resources)
- POST /api/resources/upload - Upload PDF.
- POST /api/resources/share - Share with friend.
- GET /api/resources - List files.
- GET /api/resources/:id - One file.
- DELETE /api/resources/:id - Delete file.

8. Friends & Users (/api)
- GET /api/get-friends - Friends list.
- GET /api/get-users - All users list.
- GET /api/search-users?username=... - Search users.
- GET /api/get-reqests - Received requests.
- POST /api/friend-requests/send - Body: { "recipientId" }
- POST /api/friends-request/action - Body: { "requestId", "action" }

9. Notifications (/api/get-notifications)
- GET /api/get-notifications/unseen - New alerts.
- GET /api/get-notifications/history - All alerts.

10. AI Assistant (/api/ai)
- GET /api/ai/quiz/available-options - Topics for quiz.
- POST /api/ai/quiz/generate - Make quiz.
- POST /api/ai/quiz/submit - Score quiz.
- POST /api/ai/flashcards/generate - Make flashcards.
- POST /api/ai/chat - Chat with AI.
- POST /api/ai/summary/generate - Get summary.
